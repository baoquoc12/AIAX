AI and Platform Relay Services

The GitHub repository includes example configuration files for common relay providers in the relay configuration directory, making it easier to match provider settings.

How to Use

  1. Open the repository directory: Relay configuration examples (GitHub), then read the usage notes file.
  2. Select the JSON file that matches your relay provider and import it in AI settings or copy the fields manually.
  3. Replace the api_key for every service with your own key. Update base_url if your provider gives you a custom domain.
  4. If a provider's model list differs from the example, use the provider console as the source of truth and enter the available model ID in the app.

Jimeng 2.0 / Seedream Notes

In the example configurations, the Volcengine routes for storyboard image generation (storyboard_image) and text-to-image (image) use ByteDance Seedream/Jimeng model IDs such as doubao-seedream-4-0-250828 and doubao-seedream-4-5-251128. If your relay console uses product names like "Jimeng 2.0", use the actual model ID exposed by the provider and keep it consistent in the app and JSON.

For these routes, confirm that api_protocol is volcengine and that base_url and endpoint match your relay provider's documentation. Do not mix them with OpenAI-compatible entries.

Resolution note: Seedream 4.5 and later require a minimum output pixel area. The app automatically scales small sizes up when requesting images. If a custom model or size still fails, check your relay response or use the example default model.

Jimeng 2.0 reference images: The same directory includes screenshots that help map relay product names and console fields: official Jimeng 2.0 configuration and local reverse proxy Jimeng Free API configuration.

For self-hosted Jimeng OpenAI-compatible video services, see the Jimeng AI API section in backend-node/README.md.
